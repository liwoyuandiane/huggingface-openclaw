# Long-Running Workflow（长期运行工作流）

一个用于管理长期、复杂项目的 iFlow CLI 工作流。通过增量进展和状态追踪，帮助你在多个会话中持续推进大型项目。

## 功能特性

- **初始化代理** (`/long-running-init`): 创建项目结构、进度追踪文件和功能需求列表
- **编码代理** (`/long-running-coding`): 增量完成功能，自动追踪进度
- **状态查看** (`/long-running-status`): 快速查看项目当前状态和进度概览

## 使用方法

### 1. 开始新项目

```
/long-running-init
```

然后描述你的项目需求，代理会：
- 创建 `.iflow/progress.md` 进度追踪文件
- 创建 `.iflow/features.json` 功能需求列表
- 设置 Git 初始化（如需要）
- 使用 todo_write 创建任务列表

### 2. 继续开发

```
/long-running-coding
```

每次会话开始时执行此命令，代理会：
- 读取当前进度和功能状态
- 验证环境是否正常
- 选择下一个待完成的功能
- 完成后自动更新进度并提交

### 3. 查看状态

```
/long-running-status
```

输出项目状态报告，包括：
- 功能完成进度
- 最近完成的工作
- 当前进行中的任务
- 待处理问题

## 工作原理

```
用户请求 → 初始化代理 → 创建进度文件和功能列表
     ↓
编码代理 ← 读取进度 ← 选择下一个功能
     ↓
完成功能 → 更新状态 → Git提交
     ↓
状态查看 ← 查询进度 ← 生成报告
```

## 文件结构

工作流会在项目中创建以下文件：

```
your-project/
├── .iflow/
│   ├── progress.md      # 进度追踪
│   └── features.json    # 功能需求列表
└── init.sh              # 启动脚本（可选）
```

## 功能状态说明

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `in_progress` | 进行中 |
| `completed` | 已完成（通过测试） |
| `blocked` | 被阻塞（需要外部依赖） |

## 适用场景

- 大型项目开发（需要多个会话完成）
- 功能迭代式开发
- 需要清晰进度追踪的项目
- 团队协作（通过进度文件同步状态）

## 注意事项

1. 每次会话只专注于一个功能
2. 功能完成后必须测试通过才能标记为 completed
3. 遇到阻塞时及时更新状态为 blocked
4. 保持 Git 提交记录清晰
